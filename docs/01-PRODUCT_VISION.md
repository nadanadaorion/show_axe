# Product vision

## Purpose

Ori♡n Shows centralizes the practical information required to prepare a live show without becoming a general-purpose project-management, inventory, finance, or collaboration platform.

## Primary user outcome

A user should be able to create a Show, enter it immediately, prepare equipment and people, define technical information, create an Input List, and export a technical PDF with minimal friction.

## Audience

Primary audience:

- Ori♡n and collaborators preparing live shows;
- production, stage, audio, management, and artist teams that need a shared operational snapshot.

V2 deliberately uses a shared open editor instead of personal user accounts.

## Product promise

- Fast enough to use during real production work.
- Clear enough that another person can understand a Show without training.
- Reliable offline after initial load.
- Shared between devices when online.
- Structured where structure adds value, free-form where strict taxonomies would slow the user down.

## Non-goals

Ori♡n Shows is not:

- an inventory-control system;
- an asset checkout system;
- an accounting or budgeting platform;
- a CRM;
- a ticketing system;
- a tour-management suite;
- a secure multi-tenant SaaS;
- a real-time collaborative document editor;
- a replacement for a digital mixing-console file.

## Success indicators

The product is successful when:

- a Show can be prepared without external spreadsheets;
- repeated setups can begin from Presets or earlier Shows;
- equipment readiness is visible at a glance;
- the Input List can be generated and corrected without duplicating equipment lines;
- shared devices converge on the same data without silent loss;
- public stakeholders can inspect a current Show without editing it.
